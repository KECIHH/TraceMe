# PWA、离线访问与图片压缩

TraceMe 提供基础 PWA 能力：manifest、Service Worker、离线 fallback 页面、网络状态提示、手动刷新离线摘要、清除本设备离线数据、系统/手动暗色模式和移动端适配。

## 离线能力定位

离线能力用于在旅途中临时查看低敏摘要，不用于保存完整旅行数据库、上传文件、票据附件、备份、AI prompt 或系统密钥。

用户进入具体旅行后，可以手动刷新当前旅行的离线摘要。离线摘要保存在本设备浏览器存储中，用户可手动清除。

## 缓存范围

允许缓存的摘要字段包括：

- 旅行标题、开始日期、结束日期。
- 今日行程标题、时间、地点名、交通到下一站摘要。
- 住宿名称、地址、电话。
- 交通摘要。
- 准备清单标题、分类、状态、重要性。
- 紧急备注摘要。
- 地点名称、类型、地址、电话。

默认不缓存：

- 上传文件内容。
- 备份文件。
- AI prompt 原文。
- API Key。
- session token。
- 身份证、护照、保险、订单附件内容。
- 文档加密密钥。

离线摘要接口会移除敏感字段，并对常见敏感文本做脱敏。

## 本设备离线数据

“刷新离线缓存”会把当前旅行摘要写入本设备。共享电脑、公共设备或开启浏览器同步的环境中，不建议使用离线缓存。

“清除本设备离线数据”会删除 TraceMe 离线摘要和 PWA 静态缓存。该操作只影响当前设备，不删除服务器数据。

如果未来增加“缓存敏感内容”选项，必须在启用前明确提示设备遗失、共享设备和浏览器同步风险，并提供细粒度清除能力。

## Service Worker 版本

Service Worker 缓存版本应随发布更新。实现中缓存版本来自 worker URL 的 `?v=` 参数，避免旧静态资源长期滞留。

发布 PWA 相关改动后，需要验证：

- 在线首次加载。
- 离线 fallback 页面。
- 已缓存旅行摘要的离线访问。
- 清除本设备离线数据。
- 更新后旧缓存是否被清理。

## 浏览器兼容性

- Chrome、Edge 和 Android Chrome 支持较完整。
- iOS Safari 支持添加到主屏幕，但安装提示、后台更新和存储配额行为受系统限制。
- 无痕模式、企业策略、HTTP 非本地地址或用户禁用站点存储时，离线能力可能不可用。
- PWA 安装入口和存储配额由浏览器控制，应用只能做能力检测和降级提示。

## 图片压缩

上传 JPG、PNG、WEBP 图片时，浏览器会尝试压缩大图并生成缩略图；非图片文件不会被改写。用户可以选择保留原图上传。

压缩可能降低图片质量。缩略图和上传文件仍存放在非 public 的加密上传目录中，不能通过静态 URL 直接访问。
