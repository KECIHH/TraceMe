# PWA、离线访问与图片压缩说明

TraceMe 阶段 16 提供基础 PWA 能力：manifest、Service Worker、离线 fallback 页面、网络状态提示、手动刷新离线摘要、清除本设备离线数据和系统/手动暗色模式。

## 离线缓存范围

默认只缓存当前旅行的必要摘要数据：

- 旅行标题、开始日期、结束日期；
- 今日行程标题、时间、地点名、交通到下一站摘要；
- 住宿名称、地址、电话；
- 交通摘要；
- 准备清单标题、分类、状态、重要性；
- 紧急备注摘要；
- 地点名称、类型、地址、电话。

默认不缓存上传文件内容、备份文件、AI prompt 原文、API Key、session token、身份证、护照、保险和订单附件内容。离线摘要接口会移除敏感字段并对常见敏感文本做脱敏。

## 本设备离线数据

已登录用户进入具体旅行后，可以点击“刷新离线缓存”把当前旅行摘要保存到本设备 localStorage。点击“清除本设备离线数据”会删除 TraceMe 的离线摘要和 PWA 静态缓存。

如果将来增加“缓存敏感内容”选项，必须在启用前明确提示设备遗失、共享设备和浏览器同步带来的风险。

## 浏览器兼容性

PWA 安装和 Service Worker 依赖浏览器支持。Chrome、Edge 和 Android Chrome 支持较完整；iOS Safari 支持添加到主屏幕，但安装提示、后台更新和存储配额行为受系统限制。无痕模式、企业策略、HTTP 非本地地址或用户禁用站点存储时，离线能力可能不可用。

## 图片压缩

上传 JPG、PNG、WEBP 图片时，浏览器会尝试压缩大图并生成缩略图；非图片文件不会被改写。用户可以勾选“保留原图上传”。压缩可能降低图片质量，缩略图和上传文件仍存放在非 public 的加密上传目录中。
