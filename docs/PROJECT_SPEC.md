# Project Spec

TraceMe（迹遇）是一个个人旅行规划网站，目标是在本地或私有服务器中集中管理旅行资料。它不是公开 SaaS，也不做多租户协作，核心设计目标是“自己能长期安全使用”。

## 项目目标

- 让新用户可以按 README 在本地启动、初始化管理员并体验示例旅行。
- 支持从旅行草案到出行当天的完整资料管理。
- 将行程、地点、交通、清单、预算、文件、笔记和备份放在同一套私有系统中。
- 保持数据存储简单可迁移，默认使用 SQLite。
- 提供可验证的 lint、类型检查、单元测试、E2E、构建和 Docker 部署路径。

## 功能模块

- 账号与会话：登录、退出、修改密码、会话清理、受保护页面鉴权。
- Dashboard：首页概览旅行数量、近期旅行、预算、文件、笔记和备份入口。
- 旅行计划：创建、查看、编辑、归档、删除、搜索、状态和日期管理。
- 目的地：国家、地区、时区、抵离日期、备注。
- 地点库：景点、餐厅、酒店、车站、机场、商店、医院、使领馆、交通点等类型。
- 行程日历：按旅行日期生成天数、同步缺失日期、编辑每日信息、管理行程项、排序和完成状态。
- 今日模式：根据当天或最近行程展示下一项、全部行程、住宿和文件入口。
- 路线规划：路线计划、交通方案、权重预设、评分、选择和删除。
- 准备清单：自定义清单项、状态切换、完成率、模板生成。
- 美食与住宿：基于地点扩展餐厅详情和住宿预订详情。
- 预算：分类预算、支出记录、汇率换算、已花费与剩余额度。
- 文件：上传、编辑、下载、删除、大小限制、类型校验和存储目录隔离。
- 笔记：攻略、备忘、来源链接、标签，以及 AI 结果保存为笔记。
- AI 助手：mock 或 OpenAI provider，生成草稿、敏感信息提示、历史摘要保存。
- 导出：单旅行 JSON、Markdown、HTML 导出。
- 备份：系统级 zip 备份，包含数据库快照和上传文件。
- 设置中心：个人资料、密码、AI 开关、备份管理、系统信息。
- 部署：Dockerfile、docker-compose.yml、环境变量校验、SQLite volume。

## 数据模型概览

- `User` / `Session`：管理员账号、密码 hash、登录会话。
- `Trip`：旅行主表，保存标题、状态、日期、预算、目的地摘要。
- `Destination`：旅行目的地。
- `Place`：旅行地点库，关联目的地，可扩展美食和住宿。
- `FoodDetail` / `StayDetail`：餐厅与住宿详情。
- `ItineraryDay` / `ItineraryItem`：行程日期和当天行程项。
- `RoutePlan` / `TransportOption`：路线规划和交通候选方案。
- `ChecklistItem`：准备清单。
- `CategoryBudget` / `Expense`：分类预算和支出。
- `Document`：上传文件元数据，磁盘文件在 `storage/uploads`。
- `Note`：笔记和 AI 草稿保存结果。
- `AiConversation`：AI 历史，仅保存摘要和脱敏提示。
- `AppSetting`：应用设置，如 AI 开关。
- `BackupRecord`：备份文件记录，磁盘文件在 `storage/backups`。

## 阶段说明

- 阶段 1：完成账号、旅行计划、目的地、地点、行程、清单、基础安全和测试框架。
- 阶段 2：补齐路线、交通评分、美食、住宿、预算、文件、笔记和 AI 草稿。
- 阶段 3：补齐导出、系统备份、设置中心、Docker 私有部署、移动端导航和错误页面。
- 交付阶段：完成 seed 示例数据、全量测试、文档整理、安全检查和部署验证。

## 非目标

- 不提供公开注册、多用户团队协作或权限矩阵。
- 不内置公网反向代理、HTTPS 证书申请或域名管理。
- 不自动读取上传文件交给 AI。
- 不把备份推送到第三方云盘。
