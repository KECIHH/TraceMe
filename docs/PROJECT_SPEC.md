# 项目规格

TraceMe（迹遇）是一个私有部署的 AI-first 旅行规划系统。目标是让用户输入目的地、出发城市、日期、人数、偏好和预算等少量信息，由 AI 生成结构化旅行草稿；用户预览确认后，系统把草稿写入正式旅行数据，并允许在各模块继续人工编辑。

## 产品定位

- 面向个人、家庭或可信小团队。
- 默认私有部署，强制登录。
- 支持管理员创建协作用户，不开放公众注册。
- 支持显式开启的公开分享链接，但公开视图只暴露白名单字段并过滤敏感内容。
- 支持公网域名和 HTTPS，但不默认允许搜索引擎收录。

## 核心工作流

1. 管理员通过 seed 创建初始账号。
2. 用户登录后进入 Dashboard 或旅行列表。
3. 创建旅行时优先使用 `/trips/ai-plan` 生成 AI 草稿。
4. 用户预览 AI 草稿，确认后写入 Trip、Destination、Place、Itinerary、Checklist、Budget、Route 和 Note。
5. 用户在旅行详情中继续维护行程、地点、住宿、美食、预算、文件和笔记。
6. Owner 可邀请成员、配置分享链接或导出旅行。
7. 管理员负责系统备份、用户管理和 AI Provider 配置。

## 功能模块

- 账号与会话：登录、退出、修改密码、会话管理、管理员创建用户。
- Dashboard：近期旅行、预算、文件、笔记和备份摘要。
- 旅行计划：创建、查看、编辑、归档、删除、搜索、状态和日期管理。
- AI 计划：结构化草稿生成、重新生成、丢弃、确认落库、草稿记录。
- 目的地：国家、地区、时区、抵离日期和备注。
- 地点库：景点、餐厅、酒店、车站、机场、商店、医院、使领馆、交通点等。
- 美食与住宿：推荐菜、价格、预订状态、早餐、行李寄存、取消政策等。
- 行程：按日期生成天数、管理每日信息、行程项、排序、状态和冲突提示。
- 今日模式：展示当天或最近行程、住宿和文件入口。
- 路线：路线计划、交通方案、权重预设、评分、选择和删除。
- 清单：自定义清单项、状态切换、完成率和模板生成。
- 预算：分类预算、支出记录、汇率换算和剩余额度。
- 文件：上传、加密、缩略图、下载、编辑、删除、大小限制、类型校验和非 public 存储。
- 笔记：攻略、备忘、来源链接、标签，以及 AI 结果保存为笔记。
- 导入导出：旅行 JSON / Markdown / HTML 导出，以及 CSV/JSON 导入预览和冲突处理。
- 协作：TripMember 角色、敏感文件下载授权、分享链接启用/撤销/过期/密码。
- 外部数据：地图、天气、汇率 provider，缓存和手动降级。
- PWA：manifest、Service Worker、离线摘要、网络状态和本设备离线数据清理。
- 系统设置：个人资料、密码、会话、用户、AI、备份和系统信息。

## 角色与权限

系统角色：

- `ADMIN`: 可管理系统用户、系统备份和 AI Provider 配置。
- `USER`: 可访问自己参与的旅行，不能管理系统级配置。

旅行成员角色：

- `OWNER`: 可读写旅行、删除旅行、管理成员、管理分享链接、上传文件、下载敏感文件。
- `EDITOR`: 可读写旅行和上传文件，默认不能管理成员、管理分享、删除旅行或下载敏感文件。
- `VIEWER`: 只读访问，不能上传、编辑、删除或管理成员。

可对单个成员额外授予 `canDownloadSensitiveDocuments`。

## 数据模型概览

- `User` / `Session`: 用户、系统角色、密码 hash、登录会话。
- `Trip` / `TripMember` / `TripShareLink`: 旅行、成员权限、公开分享链接。
- `Destination` / `Place` / `FoodDetail` / `StayDetail`: 目的地、地点、美食和住宿扩展信息。
- `ItineraryDay` / `ItineraryItem`: 日期和行程项。
- `RoutePlan` / `TransportOption`: 路线规划和交通候选方案。
- `ChecklistItem`: 准备清单。
- `CategoryBudget` / `Expense`: 分类预算和支出。
- `Document`: 上传文件元数据、加密元数据和缩略图元数据。
- `Note`: 笔记和保存后的 AI 结果。
- `AiPlanDraft` / `AiDraft` / `AiConversation`: AI 计划草稿、旅行内 AI 草稿和历史摘要。
- `WeatherSnapshot` / `CurrencyRate`: 外部数据快照和手动兜底。
- `ImportJob`: 导入预览、校验和结果。
- `BackupRecord` / `AuditLog` / `AppSetting`: 备份记录、审计日志和系统设置。

## 非目标

- 不做公开注册。
- 不做公开社区或多租户 SaaS。
- 不做在线支付。
- 不把上传文件、数据库或备份作为静态文件暴露。
- 不把 AI 结果当作实时票价、酒店库存、签证政策或医疗/法律/财务建议。

## 公开化前置条件

如果未来要开放给公众使用，至少需要先补齐：

- 隐私政策、用户协议和数据删除机制。
- 邮箱验证、找回密码、账号风控和滥用防护。
- 多租户隔离、配额、审计和运营后台。
- 搜索引擎策略、站点地图和公开分享合规边界。
- 第三方服务数据处理协议和日志留存策略。
